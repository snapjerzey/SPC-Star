Inspection Rules

Set the global rule used by default, then override it only where a part needs different drift behavior.

Use Global Default

Part setups can inherit the global rule selected here. If a specific part needs different drift detection, override that part in Parts & Inspections without changing the rest of the plant.

Western Electric

Uses control limits to catch classic non-random process behavior: one point beyond a control limit, two of three points near a control limit, four of five points approaching a limit, or eight consecutive points on one side of center.

Nelson Rules

Uses the current Western Electric checks and adds a Nelson-style trend signal when six consecutive measurements move in the same direction. This helps catch steady drift before parts go out of control.

CUSUM

Tracks cumulative deviation from center using a reference value of one-half sigma and an action limit of five sigma. This is sensitive to small sustained shifts that may not trip a single-point rule.

EWMA

Uses an exponentially weighted moving average with lambda 0.20 and a three-sigma EWMA limit. Recent measurements carry more weight while older process history still matters.

Moving Average Trend

Looks at the latest five measurements and compares their average to the process center. It triggers when that moving average is at least one sigma away from center.

Linear Trend / Slope

Checks the slope across the latest six measurements. It triggers when the slope is strong enough and the total movement across the window is at least one sigma.

Custom Rule

Reserved for configurable admin-defined behavior. Until the custom-rule editor is added, the default custom behavior triggers when the latest four points are beyond one sigma on the same side of center.

Spec Limit Only

Ignores process-drift patterns and locks only when a measured value is outside the configured lower or upper specification limit.

No Automatic Rule

Records inspection results without automatic drift or measured-variable lockouts. Accept/Reject failures still lock because they are direct failed inspections.

Warning Behavior

The next layer will let admins decide whether each rule creates a warning, lockout, notification, or audit-only event.
